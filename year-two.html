1.1 Importance of Algorithms & Data Structures

  • Optimizes problem-solving.
  • Improves data organization and performance.

1.2 Common Data Structures

  • Arrays: Fixed-size collections of elements.
  • Linked Lists: Nodes with data and references.
  • Stacks (LIFO): Used in function calls.
  • Queues (FIFO): Used in scheduling tasks.
  • Trees (BST): Used in hierarchical structures.
  • Graphs (Directed & Undirected): Used in networks.

1.4 Big O Notation & Time Complexity

  • O(1): Constant time.
  • O(log n): Logarithmic time.
  • O(n): Linear time.
  • O(n log n): Efficient sorting.

2.1 Types of AI

  • Narrow AI: Single-task AI (Siri, Chatbots).
  • General AI: Human-level intelligence (not yet developed).
  • Super AI: Theoretical AI beyond human intelligence.

2.2 Machine Learning (ML) & Deep Learning

  • Supervised Learning: Learns from labeled data.
  • Unsupervised Learning: Finds patterns in unlabeled data.
  • Reinforcement Learning: Learns via rewards & penalties.
  • Neural Networks: Mimic the human brain.

2.3 Natural Language Processing (NLP)

  • Enables AI to understand human language (e.g., Google Translate).

2.4 Computer Vision

  • Facial recognition, medical imaging.

3.1 Cyber Threats & Attacks

  • Malware: Viruses, Trojans, Worms.
  • Phishing: Fraudulent emails to steal data.
  • DoS Attacks: Overloading a system.
  • SQL Injection: Manipulating database queries.

3.2 Encryption & Cryptography

  • AES: Symmetric encryption.
  • RSA: Asymmetric encryption.
  • SHA-256: Secure hashing.

3.3 Network Security

  • Firewalls: Monitor network traffic.
  • VPNs: Encrypt online browsing.
  • IDS: Detects security breaches.

4.1 Software Development Life Cycle (SDLC)

  • Requirement Analysis: Understanding user needs.
  • Design: Planning architecture.
  • Implementation: Writing code.
  • Testing: Ensuring correctness.

4.2 Agile vs. Waterfall Models

  • Agile: Iterative, flexible development.
  • Waterfall: Sequential approach.

4.3 Object-Oriented Programming (OOP)

  • Encapsulation, Inheritance, Polymorphism, Abstraction.

4.4 Software Testing

  • Unit Testing: Tests individual components.
  • Integration Testing: Tests system components together.
